Managing Reports

This section covers downloading reports for a specific screening event, and for all Screenings on the Oxygen system.​

To access this page, click “Reports” on the Dashboard. This will load the ‘Reports’ page.

Downloading Reports for a Specific Screening Event​

1. Connect your computer to the Oxygen wireless network. The network name will match the label on the side of the mini-server.

2. Launch the Chrome browser on your computer. In the browser window, navigate to: https://oxygen.mystatbridge.com/

3. Log into the application, and ensure that the screening event you want to download reports from is activated.

4. When ready, click on ‘Reports’ in the Dashboard.

5. From the ‘Reports’ page, click the “Generate” button.

→ A .zip file will be downloaded to your computer. Double-click this .zip file, and the report(s) will be shown.

Note: A screening event must be active in order to download the associated reports. If no screening is active and you wish to download reports for an event, the server will not know which reports to download.

Downloading Reports for All Screenings on the Oxygen System​

1. Connect your computer to the Oxygen wireless network. The network name will match the label on the side of the mini-server.

2. Launch the Chrome browser on your computer. In the browser window, navigate to: https://oxygen.mystatbridge.com/

3. Log into the application, and click on ‘Reports’ in the Dashboard.

4. Click the “Generate All” button.

5. On the ‘Generate All Reports’ window, click the “Generate” button to confirm the action.

→ A separate .zip file will be downloaded to your computer corresponding to each screening event on the system. Double-click the .zip file for a given screening, and the reports will be shown.

Note: Any event with at least one response collected from a participant will generate a report. If a screening event was created but no participant responses were recorded, no report will be generated for that event.

Downloaded Report Contents​

When a downloaded reports .zip file is extracted, the following items will be included by default:

• disclaimerSignatures: This .zip file, when extracted, contains PNG images of each finger signature captured on the ‘Consent’ form during the screening event.

• fullExport: This .csv file includes columns for every data point collected during a screening event, and rows with data for each participant that had data collected during the screening.

• jobList: This .csv file includes columns for each data point relating to blood meter jobs, and rows for each job that was run during the screening event.

• participantList: This .csv file includes columns for data points relating to a participant’s general information, and rows for each participant who was screened during the event.

• screenerProductivity: This .csv file includes columns for data points relating to screening productivity, and rows for each screener or manager that collected data during the event.

• screeningList: This .csv file includes columns for data points relating to a screening event’s general information, and rows for each screening event that has been created on the Oxygen system.

• statkeeperBiometric: This .csv file includes columns for biometric data points collectable in a screening event, and rows for each participant who was screened during the event. This file is specifically formatted for uploading to the ‘StatKeeper’ online health portal and data storage solution.

• statkeeperEligibility: This .csv file columns for data points relating to a participant’s general information, and rows for each participant who was screened during the event. This file is specifically formatted for uploading to the ‘StatKeeper’ online health portal and data storage solution.
